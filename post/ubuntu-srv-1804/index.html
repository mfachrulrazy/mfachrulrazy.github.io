Mengenal Ubuntu Server 18.04

Ubuntu Server 18.04 Bionic Beaver

 Posted on May 23, 2018  |  2 minutes  |  MFR

Ubuntu mengeluarkan rilis LTS terbaru nya, yaitu Ubuntu 18.04 dengan codename Bionic Beaver. Banyak perubahan yang terjadi pada versi ini, berikut beberapa hal yang berbeda dari ubuntu 18.04.

Pengaturan Networking

Pengaturan IP Address

Pengaturan networking sudah tidak menggunakan tools ifupdown, tetapi menggunakan netplan. Netplan menggunakan format .yaml dengan menggunakan indentasi (seperti python).
Dalam folder /etc/netplan terdapat file 50-cloud-init.yaml di mana dalam file tersebut adalah file konfigurasi default dari ubuntu 18.04.
Untuk konfigurasi ip statik bisa mengedit file tersebut. 

# This file is generated from information provided by
# the datasource.  Changes to it will not persist across an instance.
# To disable cloud-init's network configuration capabilities, write a file
# /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg with the following:
# network: {config: disabled}
network:
    ethernets:
        enp8s0:
            dhcp4: true
        enp4s0:
            addresses: [192.168.8.2/24]
            gateway4: 192.168.8.1
            dhcp4: no
            nameservers:
                    addresses: [8.8.8.8,8.8.4.4]
    version: 2
Cara lain bisa dengan membuat file dengan nama 01-netcfg.yaml dan masukkan script network berikut jika hanya ingin mengkonfigurasi interface network enp4s0. 
network:
  version: 2
  renderer: networkd
  ethernets:
    enp4s0:
      dhcp4: no
      addresses: [192.168.8.2/24]
      gateway4: 192.168.8.1
      nameservers:
        addresses: [8.8.8.8]
Setelah itu simpan dan keluar dari edit file tersebut dan apply konfigurasi netplan dengan mode debug agar dapat melihat pesan jika ada error. 
$ sudo netplan --debug apply

Pengaturan DNS Server

Melihat konfigurasi DNS 

$ systemd-resolve --status
Konfigurasi DNS untuk statik yang tidak permanen dapat dilakukan dengan mengedit file /etc/resolv.conf dan tambahkan baris berikut 
nameserver 8.8.8.8
Atau bisa juga dengan menggunakan tools resolvconf 
$ sudo apt install resolvconf
Kemudian edit file /etc/resolvconf/resolv.conf.d/head 
nameserver 8.8.8.8
Restart Servis resolvconf 
$ sudo service resolvconf restart
