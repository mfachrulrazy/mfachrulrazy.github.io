Troubleshoot IP SSH berubah

Written by MFR with ♥ on 9 December 2018 in ssh networking remote 1 min

Ketika kita pernah me-remote server yang berbeda tetapi dengan IP yang sama menggunakan putty (ssh), maka akan muncul peringatan

Hal tersebut terjadi karena identitas server berubah (server yang berbeda).

  • Jika memilih Yes maka putty akan menggantikan identitas key ssh (fingerprint) dan menyimpan pada registri Windows.
  • Jika memilih No maka putty akan tetap melanjutkan koneksi ke server, tetapi tidak menyimpan identitas key ssh (fingerprint) di registri Windows.
  • Cancel jika tidak ingin melanjutkan koneksi.
  • Kalau tidak ingin mendapat peringatan putty, hapus terlebih dahulu ip server yang sebelumnya melalui putty cache registry di HKEY_CURRENT_USER\Software\SimonTatham\PuTTY\SshHostKeys
